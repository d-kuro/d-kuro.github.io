2019 年の振り返りと 2020 年の目標について

2019 年振り返り

SRE にジョブチェンジした

今までずっとソフトウェアエンジニアとして生きてきたんですが, SRE にジョブチェンジしました。動機としてはキャッチアップしてるうちにコンテナ界隈への興味が無限に出てきてしまったのと, ビジネスロジックを実装する人という役割に飽きてしまったというのがあります。

これからのキャリアに置いてかなり大きなターニングポイントになる気がしてますが今のところはジョブチェンジしてよかったなって思ってます。

転職した

約 1.5 年間務めたクラスメソッドを退職しました。現在は Quipper という会社で働いています。 年も明けてようやく試用期間が終わるという感じなんですが, 約 3 ヶ月間ずっとアップグレードおじさんをしていた気がします。

Quipper では Kubernetes の上にアプリケーションが乗っかっているんですが, 入社時には v1.11 が動いていたのでこれをアップグレードしていました。 あと他には Datadog agent のアップグレードとかもやってました。

Kubernetes 真面目に触り始めた

Kubernetes 真面目に触り始めたのは 2019 年でした。

めちゃくちゃ巨大でいつでもなんもわからんって感じなんですが, それでもこの 1 年でそこそこ勉強できた気がします。 転職時にもかなり重視していたポイントでこれからもどんどんやっていきたい分野です。

Kubernetes Meetup Tokyo で LT できたりもしたのもいい経験になりました。

  • 図で理解する Descheduler #k8sjp #ymju / Introduction to Descheduler
  • Argo CD 実践ガイド #k8sjp / Argo CD Practice Guide

Go 真面目に触り始めた

Go を真面目に触り始めたのも 2019 年でした。

めちゃくちゃ手に馴染んでほんとに良い言語だったのでみんなも触ってほしい。

2020 年やっていきたいこと

Kubernetes

もっとやっていきたい。

もっと Deep な部分を勉強したいという気持ちがあってコードレベルでしっかりと理解していきたい。 Kubernetes 本体へのコントリビューションはまだドキュメントぐらいしかやったことないのでしっかりとコードを貢献していけるようになりたい。

あと仕事方面でも Kubernetes 周りは無限に課題があって,

  • Kubernetes が EKS で動いていない
  • Cluster Upgrade の自動化
  • CI/CD
  • Service Mesh

あたりが大きなトピックとしてあるのでそこら辺もやっていきたい。 一緒にやっていってくれる人を募集しております。

https://career.quipper.com/jp/jobs/sre/

Go

お仕事ではもっぱら YAML 太郎なのでプライベートの時間でコードを書かないとなと思っています。

登壇

去年は LT でしか登壇をしていなかったのでもっと大きい枠での登壇をしていきたい。 大きいイベントでの CFP を出したい。

英語

転職してからナチュラルに英語でミーティングが行われる環境になったのでやっていかないといけない。 現段階では中学英語ができてないのでやばい。 Google 翻訳に頼り切って日々を過ごしています。

英語でブログ書いたりするものいいかもしれない。

読書

去年いっぱい本買って積んだので定期的に読書していきたい。 一ヶ月一冊くらい読みたい。
